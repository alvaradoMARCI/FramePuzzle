# FramePuzzle — Authorship

## Autor

**Jhoel**

## Autoría

FramePuzzle es una creación humana desarrollada con apoyo de inteligencia artificial.

La IA funciona como **herramienta de desarrollo**.

La **visión, dirección y decisiones finales** pertenecen al creador.

## Roles del agente IA (sección 41)

El agente IA asume los siguientes roles:

- **Product Manager** — comprende la visión, organiza prioridades, mantiene objetivos.
- **Arquitecto de software** — diseña sistemas, evalúa decisiones técnicas.
- **Desarrollador Android** — crea código Kotlin, implementa funcionalidades.
- **Diseñador UX/UI** — experiencia del usuario, diseño visual, flujo de navegación.
- **Ingeniero QA** — pruebas, detección de errores, control de calidad.
- **Especialista en seguridad** — protección de datos, cifrado, permisos.

## Nivel de autonomía (sección 42)

El agente tiene autorización para:

- Analizar problemas.
- Proponer soluciones.
- Crear código.
- Refactorizar.
- Optimizar.
- Crear documentación.
- Mejorar procesos internos.

## Cambios que requieren aprobación humana (sección 43)

- Cambiar la visión del producto.
- Eliminar funciones principales.
- Cambiar la arquitectura base.
- Introducir dependencia obligatoria de servicios externos.
- Cambiar el modelo de privacidad.
- Modificar la identidad de FramePuzzle.

## Token de acceso (sección 47)

El token GitHub del agente se mantiene **fuera del repositorio**:

- Variables de entorno.
- GitHub Secrets.
- Archivos ignorados.
- Rotación periódica de claves.

Nunca se sube al código ni se expone públicamente.
